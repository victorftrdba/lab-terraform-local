# CLAUDE.md - lab-terraform-local

## What it is

Terraform lab for provisioning local AWS infrastructure via LocalStack (Docker), with no external costs. Creates S3 (Data Lake) and SQS (queues) as the foundation for a complete ETL pipeline in a 100% isolated environment.

## Stack

- Terraform (HCL)
- LocalStack (local AWS emulation, port 4566)
- AWS CLI / awslocal (AWSLocal CLI)
- Mailpit (test email capture, Docker)
- Docker Compose (orchestrates LocalStack and Mailpit)

## Structure

```
main.tf              -> Main configuration of the AWS resources
variables.tf         -> Variable declarations
outputs.tf           -> Outputs of the provisioned resources
terraform.tfvars     -> Variable values (do not commit secrets)
terraform.tfstate    -> Local state generated by terraform apply
modules/s3/          -> Terraform module for S3 buckets (Data Lake)
modules/sqs/         -> Terraform module for SQS queues
```

## Commands

```bash
terraform init               # Initialize providers
terraform plan               # Preview changes
terraform apply              # Provision infrastructure on LocalStack
terraform destroy            # Destroy resources
```

## Conventions

- Provider points to `http://localhost:4566` (LocalStack), never to real AWS
- Modules isolated per service (s3/, sqs/) with explicit inputs/outputs
- `terraform.tfvars` defines the local endpoint; do not commit real credentials
- Lab integrated with pipeline-etl-localstack and data-validator-app (external repos)
